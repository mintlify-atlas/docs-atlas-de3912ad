title: "Fabric Gateway"
description: "The Fabric Gateway service simplifies client application development by delegating endorsement gathering and transaction submission to the peer."
---

Fabric Gateway is a service embedded in every Fabric v2.4+ peer that provides a simplified API for submitting transactions. Rather than requiring client SDKs to contact multiple peers for endorsements, the gateway peer handles the entire endorsement workflow on behalf of the client.

## How the Gateway works

The gateway manages five transaction operations:

| Operation | Description |
|-----------|-------------|
| **Evaluate** | Simulate a transaction on a single peer (read/query — no ledger update) |
| **Endorse** | Gather endorsements from required organizations per the endorsement policy |
| **Submit** | Send the signed endorsed transaction to the ordering service |
| **CommitStatus** | Wait for the transaction to be committed to the ledger |
| **ChaincodeEvents** | Receive events emitted by chaincode when transactions commit |

The `SubmitTransaction` convenience method combines Endorse + Submit + CommitStatus into a single blocking call.

## Peer selection

The gateway selects endorsing peers automatically using the discovery service:

1. It prefers peers in the same organization as the gateway peer (trusted by the client)
2. Among eligible peers, it selects the one with the **highest block height**
3. If an organization's peer is unavailable, the gateway retries with another qualifying peer
4. The gateway derives the endorsement plan from the `ChaincodeInterest` structure — it handles policy combinations including state-based endorsement (SBE) policies automatically

<Note>
For private data (transient data), the gateway restricts endorsing organizations to those that are members of the relevant private data collection, to prevent sensitive data leakage.
</Note>

## Connecting to the Gateway

<Tabs>
  <Tab title="Go">
    ```go
    import (
        "crypto/x509"
        "github.com/hyperledger/fabric-gateway/pkg/client"
        "github.com/hyperledger/fabric-gateway/pkg/identity"
        "google.golang.org/grpc"
        "google.golang.org/grpc/credentials"
    )

    // Load TLS certificate
    certificate, _ := loadCertificate("path/to/peer/tls/ca.crt")
    certPool := x509.NewCertPool()
    certPool.AddCert(certificate)
    transportCredentials := credentials.NewClientTLSFromCert(certPool, "")

    // Create gRPC connection
    connection, _ := grpc.NewClient(
        "peer0.org1.example.com:7051",
        grpc.WithTransportCredentials(transportCredentials),
    )
    defer connection.Close()

    // Create identity and signing implementation
    id, _ := identity.NewX509Identity("Org1MSP", certificate)
    sign, _ := identity.NewPrivateKeySign(privateKey)

    // Connect to gateway
    gw, _ := client.Connect(id, client.WithSign(sign), client.WithClientConnection(connection))
    defer gw.Close()

    network := gw.GetNetwork("mychannel")
    contract := network.GetContract("basic")
    ```
  </Tab>
  <Tab title="Node.js">
    ```typescript
    import * as grpc from '@grpc/grpc-js';
    import { connect, hash, signers } from '@hyperledger/fabric-gateway';
    import * as crypto from 'node:crypto';
    import { promises as fs } from 'node:fs';

    const client = new grpc.Client(
        'peer0.org1.example.com:7051',
        grpc.credentials.createSsl(await fs.readFile('path/to/peer/tls/ca.crt'))
    );

    const gateway = connect({
        client,
        identity: { mspId: 'Org1MSP', credentials: await fs.readFile('path/to/cert.pem') },
        signer: signers.newPrivateKeySigner(
            crypto.createPrivateKey(await fs.readFile('path/to/key.pem'))
        ),
    });

    const network = gateway.getNetwork('mychannel');
    const contract = network.getContract('basic');
    ```
  </Tab>
  <Tab title="Java">
    ```java
    import org.hyperledger.fabric.client.*;
    import io.grpc.ManagedChannel;
    import io.grpc.netty.shaded.io.grpc.netty.GrpcSslContexts;
    import io.grpc.netty.shaded.io.grpc.netty.NettyChannelBuilder;

    ManagedChannel channel = NettyChannelBuilder
        .forAddress("peer0.org1.example.com", 7051)
        .sslContext(GrpcSslContexts.forClient()
            .trustManager(Paths.get("path/to/peer/tls/ca.crt").toFile())
            .build())
        .build();

    Gateway gateway = Gateway.newInstance()
        .identity(Identities.newX509Identity("Org1MSP", certificate))
        .signer(Signers.newPrivateKeySigner(privateKey))
        .connection(channel)
        .connect();

    Network network = gateway.getNetwork("mychannel");
    Contract contract = network.getContract("basic");
    ```
  </Tab>
</Tabs>

## Submitting transactions

`SubmitTransaction` — synchronous, waits for commit:

<Tabs>
  <Tab title="Go">
    ```go
    result, err := contract.SubmitTransaction("CreateAsset", "asset1", "blue", "10", "Alice", "1300")
    if err != nil {
        panic(fmt.Errorf("failed to submit transaction: %w", err))
    }
    fmt.Printf("Result: %s\n", result)
    ```
  </Tab>
  <Tab title="Node.js">
    ```typescript
    const result = await contract.submitTransaction('CreateAsset', 'asset1', 'blue', '10', 'Alice', '1300');
    console.log('Result:', Buffer.from(result).toString());
    ```
  </Tab>
  <Tab title="Java">
    ```java
    byte[] result = contract.submitTransaction("CreateAsset", "asset1", "blue", "10", "Alice", "1300");
    System.out.println("Result: " + new String(result));
    ```
  </Tab>
</Tabs>

## Chaincode events

Listen for events emitted by chaincode:

<Tabs>
  <Tab title="Go">
    ```go
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()

    events, err := network.ChaincodeEvents(ctx, "basic")
    if err != nil {
        panic(err)
    }

    for event := range events {
        fmt.Printf("Event: %s, TxID: %s, Payload: %s\n",
            event.EventName, event.TransactionID, event.Payload)
    }
    ```
  </Tab>
  <Tab title="Node.js">
    ```typescript
    const events = await network.getChaincodeEvents('basic');
    for await (const event of events) {
        console.log(`Event: ${event.eventName}, TxID: ${event.transactionId}`);
    }
    ```
  </Tab>
</Tabs>

## Error handling

The gateway returns structured errors that include details about which peer or orderer failed:

```go
if err != nil {
    var endorseErr *client.EndorseError
    var submitErr *client.SubmitError
    var commitErr *client.CommitError

    switch {
    case errors.As(err, &endorseErr):
        fmt.Printf("Endorse error for tx %s: %v\n", endorseErr.TransactionID, endorseErr)
    case errors.As(err, &submitErr):
        fmt.Printf("Submit error for tx %s: %v\n", submitErr.TransactionID, submitErr)
    case errors.As(err, &commitErr):
        fmt.Printf("Commit error for tx %s with code %d: %v\n",
            commitErr.TransactionID, int32(commitErr.Code), commitErr)
    default:
        fmt.Printf("Unexpected error: %v\n", err)
    }
}
```

## Timeout configuration

Configure timeouts in `core.yaml` under the `peer.gateway` section:

```yaml
peer:
  gateway:
    # Timeout for collecting endorsements from external peers
    endorsementTimeout: 30s
    # Timeout for broadcasting to orderer nodes
    broadcastTimeout: 30s
```

<Tip>
Client applications can also set per-call deadlines using `context.WithTimeout` (Go) or gRPC call options (Node.js/Java) for fine-grained control over individual operations.
</Tip>
