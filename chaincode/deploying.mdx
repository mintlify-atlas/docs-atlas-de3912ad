---
title: "Deploying Chaincode"
description: "Step-by-step guide to packaging, installing, and committing chaincode to a Fabric channel."
---

This tutorial walks through deploying the asset-transfer (basic) chaincode to a running Fabric network using the `peer lifecycle chaincode` commands. By the end, you will have a working chaincode that you can invoke and query.

<Note>
This tutorial uses the Fabric test network from fabric-samples. Make sure you have completed the [install](/getting-started/install) steps before continuing.
</Note>

## Prerequisites

Start the test network with a channel:

```bash
cd fabric-samples/test-network
./network.sh down
./network.sh up createChannel
```

## Environment setup

You will need separate terminal sessions (or environment variable switching) for each org. Set these variables for **Org1** to start:

```bash
export PATH=${PWD}/../bin:$PATH
export FABRIC_CFG_PATH=$PWD/../config/

export CORE_PEER_TLS_ENABLED=true
export CORE_PEER_LOCALMSPID="Org1MSP"
export CORE_PEER_TLS_ROOTCERT_FILE=${PWD}/organizations/peerOrganizations/org1.example.com/peers/peer0.org1.example.com/tls/ca.crt
export CORE_PEER_MSPCONFIGPATH=${PWD}/organizations/peerOrganizations/org1.example.com/users/Admin@org1.example.com/msp
export CORE_PEER_ADDRESS=localhost:7051
```

## Deployment steps

<Steps>
  <Step title="Package the chaincode">
    Navigate to the chaincode directory and fetch dependencies (Go example):

    ```bash
    cd fabric-samples/asset-transfer-basic/chaincode-go
    GO111MODULE=on go mod vendor
    cd ../../test-network
    ```

    Package the chaincode:

    ```bash
    peer lifecycle chaincode package basic.tar.gz \
      --path ../asset-transfer-basic/chaincode-go/ \
      --lang golang \
      --label basic_1.0
    ```

    This creates `basic.tar.gz` — a deployable package containing your chaincode and its metadata.
  </Step>

  <Step title="Install on Org1 peer">
    With the Org1 environment variables set:

    ```bash
    peer lifecycle chaincode install basic.tar.gz
    ```

    Expected output:
    ```
    Chaincode code package identifier: basic_1.0:4ec191e0b0f...
    ```

    Note the package ID — you will need it in later steps.
  </Step>

  <Step title="Install on Org2 peer">
    Switch environment variables to Org2:

    ```bash
    export CORE_PEER_LOCALMSPID="Org2MSP"
    export CORE_PEER_TLS_ROOTCERT_FILE=${PWD}/organizations/peerOrganizations/org2.example.com/peers/peer0.org2.example.com/tls/ca.crt
    export CORE_PEER_MSPCONFIGPATH=${PWD}/organizations/peerOrganizations/org2.example.com/users/Admin@org2.example.com/msp
    export CORE_PEER_ADDRESS=localhost:9051

    peer lifecycle chaincode install basic.tar.gz
    ```
  </Step>

  <Step title="Query the installed package ID">
    ```bash
    peer lifecycle chaincode queryinstalled
    ```

    Output:
    ```
    Installed chaincodes on peer:
    Package ID: basic_1.0:4ec191e0b0f..., Label: basic_1.0
    ```

    Save the package ID to a variable:

    ```bash
    export CC_PACKAGE_ID=basic_1.0:4ec191e0b0f...
    ```
  </Step>

  <Step title="Approve for Org2">
    Still using Org2 environment:

    ```bash
    peer lifecycle chaincode approveformyorg \
      -o localhost:7050 \
      --ordererTLSHostnameOverride orderer.example.com \
      --channelID mychannel \
      --name basic \
      --version 1.0 \
      --package-id $CC_PACKAGE_ID \
      --sequence 1 \
      --tls \
      --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem"
    ```
  </Step>

  <Step title="Approve for Org1">
    Switch back to Org1 environment variables, then:

    ```bash
    peer lifecycle chaincode approveformyorg \
      -o localhost:7050 \
      --ordererTLSHostnameOverride orderer.example.com \
      --channelID mychannel \
      --name basic \
      --version 1.0 \
      --package-id $CC_PACKAGE_ID \
      --sequence 1 \
      --tls \
      --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem"
    ```
  </Step>

  <Step title="Check commit readiness">
    Verify both orgs have approved before committing:

    ```bash
    peer lifecycle chaincode checkcommitreadiness \
      --channelID mychannel \
      --name basic \
      --version 1.0 \
      --sequence 1 \
      --tls \
      --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem" \
      --output json
    ```

    Expected output:
    ```json
    {
      "approvals": {
        "Org1MSP": true,
        "Org2MSP": true
      }
    }
    ```
  </Step>

  <Step title="Commit the chaincode definition">
    ```bash
    peer lifecycle chaincode commit \
      -o localhost:7050 \
      --ordererTLSHostnameOverride orderer.example.com \
      --channelID mychannel \
      --name basic \
      --version 1.0 \
      --sequence 1 \
      --tls \
      --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem" \
      --peerAddresses localhost:7051 \
      --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org1.example.com/peers/peer0.org1.example.com/tls/ca.crt" \
      --peerAddresses localhost:9051 \
      --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org2.example.com/peers/peer0.org2.example.com/tls/ca.crt"
    ```
  </Step>

  <Step title="Verify deployment">
    ```bash
    peer lifecycle chaincode querycommitted \
      --channelID mychannel \
      --name basic \
      --tls \
      --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem"
    ```

    Output:
    ```
    Committed chaincode definition for chaincode 'basic' on channel 'mychannel':
    Version: 1.0, Sequence: 1, Endorsement Plugin: escc, Validation Plugin: vscc,
    Approvals: [Org1MSP: true, Org2MSP: true]
    ```
  </Step>
</Steps>

## Invoke and query

Initialize the ledger, then query it:

```bash
peer chaincode invoke \
  -o localhost:7050 \
  --ordererTLSHostnameOverride orderer.example.com \
  --tls \
  --cafile "${PWD}/organizations/ordererOrganizations/example.com/orderers/orderer.example.com/msp/tlscacerts/tlsca.example.com-cert.pem" \
  -C mychannel \
  -n basic \
  --peerAddresses localhost:7051 \
  --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org1.example.com/peers/peer0.org1.example.com/tls/ca.crt" \
  --peerAddresses localhost:9051 \
  --tlsRootCertFiles "${PWD}/organizations/peerOrganizations/org2.example.com/peers/peer0.org2.example.com/tls/ca.crt" \
  -c '{"function":"InitLedger","Args":[]}'
```

```bash
peer chaincode query -C mychannel -n basic -c '{"Args":["GetAllAssets"]}'
```

## Upgrading chaincode

To upgrade, increment the `--sequence` number and repeat the approve/commit steps with an updated `--version`:

```bash
peer lifecycle chaincode approveformyorg ... --version 2.0 --sequence 2
peer lifecycle chaincode commit         ... --version 2.0 --sequence 2
```

<Tip>
You do not need to re-install the chaincode package on peers unless the package itself changed. Changing only the endorsement policy or initialization parameters requires incrementing the sequence but not re-installing.
</Tip>
